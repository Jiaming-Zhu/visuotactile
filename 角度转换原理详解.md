# 真实机械臂角度转换原理详解

## 🎯 核心答案

**是的！真实机械臂的角度转换完全依赖于 `main_follower.json` 校准文件。**

这个文件定义了：
- 每个舵机的零点位置（`homing_offset`）
- 旋转方向（`drive_mode`）
- 校准模式（`calib_mode`）

---

## 📋 校准文件内容解析

### 当前的 `main_follower.json`：

```json
{
  "homing_offset": [-2049, 3044, -1010, -2070, 2086, -2046],
  "drive_mode": [0, 1, 0, 0, 1, 0],
  "start_pos": [2010, 2993, 1113, 2024, 2047, 1893],
  "end_pos": [3073, -2020, 2034, 3094, -1062, 3070],
  "calib_mode": ["DEGREE", "DEGREE", "DEGREE", "DEGREE", "DEGREE", "LINEAR"],
  "motor_names": ["shoulder_pan", "shoulder_lift", "elbow_flex", "wrist_flex", "wrist_roll", "gripper"]
}
```

### 各字段含义：

| 字段 | 作用 | 示例 |
|------|------|------|
| `homing_offset` | 定义零点位置（步进值偏移） | `-2049` 表示零点在步进值 -2049 |
| `drive_mode` | 旋转方向（0=正向，1=反向） | `1` 表示反转该关节的旋转方向 |
| `calib_mode` | 校准模式（DEGREE 或 LINEAR） | `DEGREE` 用于旋转关节，`LINEAR` 用于夹爪 |
| `start_pos` / `end_pos` | 线性关节的起止位置 | 仅用于夹爪等线性运动 |
| `motor_names` | 舵机名称列表 | 对应6个电机 |

## 🔄 角度转换流程详解

### 从你的程序发送角度 → 舵机实际位置

当你在 `interactive_control.py` 中发送 `45°` 到舵机时，经历以下转换：

#### **步骤1：你的代码（度数）**
```python
joint_angles_deg = 45.0  # 度数
```

#### **步骤2：LeRobot 的 `revert_calibration()` 自动转换**

位于 `feetech.py` 第 837-894 行，执行以下转换：

```python
def revert_calibration(values, motor_names):
    # 以 shoulder_pan 为例（索引 0）
    # homing_offset = -2049
    # drive_mode = 0
    # resolution = 4096 (舵机分辨率)
    
    # 步骤 2.1: 角度 → 步进值
    # 45° → 512 步
    step_value = 45.0 / 180.0 * (4096 // 2)  # = 512
    
    # 步骤 2.2: 应用零点偏移
    # 512 → 512 - (-2049) = 2561
    step_value -= homing_offset  # = 512 - (-2049) = 2561
    
    # 步骤 2.3: 应用旋转方向
    if drive_mode == 1:  # 如果是 shoulder_lift (drive_mode=1)
        step_value *= -1  # 反转
    
    # 最终发送给舵机: 2561
    return int(step_value)
```

#### **步骤3：舵机接收步进值**
```
舵机收到：2561（原始步进值）
舵机实际旋转到：步进位置 2561
```

当你使用 `compare` 命令读取真实机械臂角度时：

#### **步骤1：从舵机读取**
```python
present_pos = motors.read("Present_Position")
# 舵机返回原始步进值：2561
```

#### **步骤2：LeRobot 的 `apply_calibration()` 自动转换**

位于 `feetech.py` 第 648-728 行：

```python
def apply_calibration(values, motor_names):
    # 以 shoulder_pan 为例
    # homing_offset = -2049
    # drive_mode = 0
    # resolution = 4096
    
    raw_value = 2561  # 舵机返回的原始值
    
    # 步骤 2.1: 应用旋转方向
    if drive_mode == 1:
        raw_value *= -1
    
    # 步骤 2.2: 应用零点偏移
    # 2561 → 2561 + (-2049) = 512
    raw_value += homing_offset  # = 512
    
    # 步骤 2.3: 步进值 → 角度
    # 512 → 45°
    angle_deg = raw_value / (4096 // 2) * 180.0  # = 45.0°
    
    return angle_deg
```

#### **步骤3：显示给你**
```
真实机械臂角度: 45.00°
```

## 🔑 关键概念

### 1. **零点偏移 (`homing_offset`)**

- **问题**：不同舵机的"出厂零点"位置不同
  - 舵机 A 的零点可能在步进值 2048
  - 舵机 B 的零点可能在步进值 56734
  
- **解决**：通过 `homing_offset` 统一零点
  - 例如 `homing_offset = -2049` 意味着：
    - 舵机物理零点在步进值 `-2049`
    - 当发送 `0°` 时，实际发送步进值 `-2049`

### 2. **旋转方向 (`drive_mode`)**

- **问题**：主从臂可能装配方向相反
  - Leader 臂顺时针 = Follower 臂逆时针
  
- **解决**：通过 `drive_mode` 反转方向
  - `drive_mode = 0`：正常方向
  - `drive_mode = 1`：反转方向
  
- **示例**（`shoulder_lift`）：
  - 你发送 `45°`
  - `drive_mode = 1`，转换为 `-45°`
  - 实现主从臂镜像动作

### 3. **舵机分辨率 (`resolution`)**

- **定义**：舵机一圈（360°）有多少个步进位置
  - Feetech 舵机：通常 4096 步
  - Dynamixel 舵机：4096 步
  
- **转换公式**：
  ```
  步进值 = 角度 / 180° × (分辨率 / 2)
  ```
  
  例如对于 4096 分辨率：
  - 90° = 1024 步
  - 45° = 512 步
  - 180° = 2048 步

## 📊 完整转换示例

### 场景：控制 `shoulder_lift` 到 `-45°`

| 阶段 | 值 | 说明 |
|------|-----|------|
| 🟢 你的代码 | `-45.0°` | 你希望的角度 |
| 🔹 转换为步进值 | `-45 / 180 × 2048 = -512` | 角度→步进值 |
| 🔹 移除零点偏移 | `-512 - 3044 = -3556` | 减去 homing_offset |
| 🔹 应用旋转方向 | `-3556 × (-1) = 3556` | drive_mode=1 反转 |
| 🔴 发送到舵机 | `3556` | 舵机接收的原始值 |

## 🎯 为什么需要校准文件？

### 没有校准文件的问题：

```python
# ❌ 不同舵机的"零点"不同
motor1.write(0)    # 舵机1: 步进值 0
motor2.write(0)    # 舵机2: 步进值 0
# 但两个舵机的物理位置可能完全不同！
```

### 有校准文件的优势：

```python
# ✅ 统一的坐标系
motors.write("Goal_Position", 0.0)  # 度数
# 自动转换为各个舵机的正确步进值
# motor1 → -2049
# motor2 → 3044
# 所有舵机都回到物理零点！
```

## 🛠️ 如何生成校准文件？

### 方法1：自动校准（推荐）

```bash
cd lerobot
python lerobot/scripts/control_robot.py calibrate \
  --robot-path lerobot/configs/robot/so101.yaml
```

这会：
1. 提示你手动将机械臂移动到"零位姿态"
2. 记录每个舵机此时的步进值
3. 计算 `homing_offset`
4. 保存到 `.cache/calibration/so101/main_follower.json`

### 方法2：手动修改

如果你发现某个关节的零点不对：

```json
{
  "homing_offset": [-2049, 3044, -1010, -2070, 2086, -2046],
                           ^^^^
                           调整这个值
}
```

## 📝 总结

### 关键要点：

1. ✅ **所有角度转换都依赖于 `main_follower.json`**
2. ✅ **LeRobot 自动处理转换，你只需要使用度数**
3. ✅ **校准文件统一了不同舵机的坐标系**
4. ✅ **三个关键参数**：
   - `homing_offset`：零点位置
   - `drive_mode`：旋转方向
   - `resolution`：舵机分辨率（通常是 4096）

### 数据流：

```
你的代码（度数）
    ↓
interactive_control.py
    ↓
LeRobot write() 
    ↓
revert_calibration() ← 使用 main_follower.json
    ↓
舵机（步进值 0-4095）
```

### 最重要的一点：

**校准文件定义了"什么是零度"！**

没有它，你发送的 `0°` 可能让舵机转到任何位置。
有了它，`0°` 就是你校准时定义的那个物理位置。
