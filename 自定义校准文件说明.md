## 🔑 关键知识点

### 1. `calibration_dir` 是目录路径

**❌ 错误写法**：
```python
calibration_dir="./main_follower.json"  # 文件路径，LeRobot 会出错
```

**✅ 正确写法**：
```python
calibration_dir="/home/martina/Y3_Project/learn_PyBullet"  # 目录路径
```

**原理**：
- LeRobot 内部会在目录下查找 `main_follower.json`
- 代码逻辑（简化）：
  ```python
  calibration_file = f"{calibration_dir}/main_follower.json"
  ```

---

### 2. 为什么需要自定义校准文件？

**原因1：零点不对应**
- 原始校准文件是在 LeRobot 官方机械臂上生成的
- 你的机械臂组装角度、舵机零点可能不同
- 需要根据实际情况调整

**原因2：不同的零点定义**
- PyBullet URDF 的零点姿势
- LeRobot 校准的零点姿势
- 可能不完全一致

## 🔄 如何更新校准值？

### 情况1：通过 raw 命令采集数据

```bash
1. 运行 raw 命令记录零点
2. 查看 saved_raw_positions_*.json 中的 raw_steps
3. 计算新的 homing_offset = 0 - raw_steps
4. 更新 main_follower.json
```

**示例**：

假设你在 raw 模式下记录的零位步进值是：
```json
"raw_steps": [2049, 2005, 2084, 2070, -2086, 1887]
```

计算 homing_offset：
```python
homing_offset = [
    0 - 2049,   # = -2049
    0 - 2005,   # = -2005
    0 - 2084,   # = -2084
    0 - 2070,   # = -2070
    0 - (-2086), # = 2086
    0 - 1887    # = -1887
]
# 结果：[-2049, -2005, -2084, -2070, 2086, -1887]
```

### 情况2：微调单个关节

如果发现某个关节有偏差：

```bash
# 例如：关节2偏差5度
# 原来：homing_offset[2] = -2084
# 调整：homing_offset[2] = -2084 - (5/180*2048) ≈ -2084 - 57 = -2141
```

## 💡 常见问题

### Q1: 为什么 shoulder_lift 的 drive_mode 是 1？

**A**: `drive_mode = 1` 表示该关节的旋转方向反转。
- 可能是机械臂装配时该关节方向相反
- 或者为了实现主从臂镜像控制

### Q2: 为什么我的 homing_offset 和原始不同？

**A**: 完全正常！原因：
1. 你的机械臂装配角度可能不同
2. 舵机的物理零点不同
3. URDF 零点定义可能不同

### Q3: 如何知道我的校准是否正确？

**A**: 测试方法：
```bash
> joint 0 0 0 0 0    # 回零
# 如果机械臂呈现"水平伸直+夹爪朝上" → 正确
# 如果不是 → 需要调整
```

### Q4: 相对路径可以吗？

**A**: 可以，但不推荐：
```python
# ✅ 推荐：绝对路径
calibration_dir="/home/martina/Y3_Project/learn_PyBullet"

# ⚠️  可以但不推荐：相对路径（取决于运行目录）
calibration_dir="./learn_PyBullet"
```
