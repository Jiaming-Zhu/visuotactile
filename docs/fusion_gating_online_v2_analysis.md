## 1. 版本定位

`fusion_gating_online_v2` 不是一个全新的网络骨架，而是：

> `train_fusion_gating2.py` 中 `FusionModel` 的前缀在线训练版

它的核心思路是：

1. 保留原始的视觉-触觉门控融合结构；
2. 训练时随机截断触觉前缀，让模型不总是看到完整触觉序列；
3. 评估时显式测量 `10% / 20% / 40% / ... / 100%` 前缀下的预测准确率；
4. 让模型具备“抓取进行到一部分时就尝试做判断”的能力。

因此，这一版的“在线”含义是：

- 可以在抓取进行到 `n%` 时，用当前已经观测到的触觉前缀做预测；
- 每次预测都会用“当前前缀整段”重新跑一次模型；
- 它不是严格的增量缓存式 streaming model，没有 `forward_step + state cache`。

从部署角度看，它属于：

- **功能上可在线预测**
- **实现上非增量式在线**

## 2. 模型结构

### 2.1 视觉分支

- Backbone：ImageNet 预训练 `ResNet18`
- 投影：`1x1 Conv2d(512 -> 256)`
- 输出：`49` 个视觉 token
- 冻结策略：`freeze_visual=True`

### 2.2 触觉分支

- 输入：`24` 通道触觉时间序列
- 编码器：3 层 `Conv1d + BatchNorm1d + ReLU`
- 每层 `stride=2`
- 作用：将原始触觉序列压缩为触觉 token 序列

### 2.3 门控融合

脚本复用 `train_fusion_gating2.py` 中的门控机制：

- 从视觉全局摘要 `v_global` 和触觉全局摘要 `t_global` 估计门控分数 `g`
- 用 `g` 缩放视觉 token
- 当 `g` 较低时，视觉信息被部分替换为一个可学习的“无信息先验” `t_null`

因此，`g` 不是最终决策权重，而是：

- 对视觉 token 的连续抑制系数

### 2.4 融合主干

- 序列拼接方式：`[CLS] + gated visual tokens + tactile tokens`
- 主干：4 层 Transformer Encoder
- 输出：从 `[CLS]` token 读取三个任务分类结果

### 2.5 辅助头

除主任务头外，还保留 3 个触觉辅助头：

- `aux_mass`
- `aux_stiffness`
- `aux_material`

它们的作用是防止模型完全依赖视觉捷径，迫使触觉分支保持可分性。

## 3. 训练方式与关键超参数

本次 `fusion_gating_online_v2` 的 checkpoint 配置为：

| 参数 | 值 |
| --- | --- |
| `epochs` | `60` |
| `batch_size` | `32` |
| `lr` | `1e-4` |
| `weight_decay` | `0.01` |
| `warmup_epochs` | `5` |
| `fusion_dim` | `256` |
| `num_heads` | `8` |
| `num_layers` | `4` |
| `dropout` | `0.1` |
| `freeze_visual` | `True` |
| `visual_drop_prob` | `0.1` |
| `tactile_drop_prob` | `0.0` |
| `lambda_reg` | `0.1` |
| `lambda_aux` | `0.5` |
| `reg_type` | `entropy` |
| `gate_reg_warmup_epochs` | `5` |
| `gate_reg_ramp_epochs` | `10` |
| `online_train_prob` | `0.6` |
| `online_min_prefix_ratio` | `0.4` |
| `min_prefix_len` | `64` |

这组超参数的关键含义是：

- 不是所有训练样本都做前缀截断，`40%` 左右的最短前缀作为在线训练下限；
- gate 正则用 `entropy`，目标不是把 `g` 推向极值，而是避免过早饱和；
- 训练中加入少量视觉 dropout，降低视觉捷径风险。

这也引入了一个很明确的训练偏差：

- 只有约 `60%` 的训练样本会使用前缀训练；
- 一旦使用前缀，前缀长度也基本落在 `40%-100%` 区间；
- 因此模型在训练过程中很少，甚至几乎不会看到 `40%` 以下的超短触觉前缀。

这会直接影响极早期在线预测能力。

## 5. 模态诊断

### 5.1 In-Distribution (`test`)

| 模式 | Avg Acc |
| --- | ---: |
| 正常融合 | `100.00%` |
| 屏蔽视觉，`tactile-only` | `92.65%` |
| 屏蔽触觉，`visual-only` | `50.98%` |

解读：

- 在 ID 上，视觉很有帮助；
- 但触觉单独也已经很强；
- 视觉单独只能做到中等偏弱，说明最终成功并不是纯视觉捷径。

### 5.2 OOD (`ood_test`)

| 模式 | Avg Acc |
| --- | ---: |
| 正常融合 | `99.17%` |
| 屏蔽视觉，`tactile-only` | `94.44%` |
| 屏蔽触觉，`visual-only` | `32.50%` |

解读：

- OOD 时，真正可靠的是触觉；
- 视觉单独几乎失效；
- 融合结果比 `tactile-only` 还略高，说明视觉并没有明显拖后腿；
- 这是一个“以触觉为主、视觉为辅”的健康融合状态。

这点和失败模型有显著区别：

- 失败模型通常表现为：`visual-only` 很强，`tactile-only` 很弱；
- 或者 OOD 下屏蔽视觉后结果反而更好。

而 `fusion_gating_online_v2` 恰恰没有出现这个问题。

## 6. 在线前缀曲线

OOD 在线曲线如下：

| Prefix Ratio | Mass | Stiffness | Material | Avg Acc | Avg Gate |
| --- | ---: | ---: | ---: | ---: | ---: |
| `0.1` | `42.50%` | `25.00%` | `25.00%` | `30.83%` | `0.7104` |
| `0.2` | `50.83%` | `26.67%` | `26.67%` | `34.72%` | `0.7250` |
| `0.4` | `90.00%` | `91.67%` | `90.00%` | `90.56%` | `0.7226` |
| `0.6` | `100.00%` | `99.17%` | `100.00%` | `99.72%` | `0.6402` |
| `0.8` | `100.00%` | `98.33%` | `100.00%` | `99.44%` | `0.6454` |
| `1.0` | `100.00%` | `98.33%` | `99.17%` | `99.17%` | `0.6506` |

### 6.1 这条曲线说明了什么

这条曲线呈现出非常典型的“中段收敛”行为：

- `10%-20%` 前缀时，模型还不够强；
- `40%` 前缀时，准确率已经超过 `90%`；
- `60%` 前缀时，基本完成决策；
- `80%-100%` 区间进入稳定平台期。

因此它更像一个：

- **中段可用**
- **后段高置信收敛**

的在线预测器，而不是一个“极早期就能准确判断”的模型。

### 6.2 极早期性能断崖的原因

`10%-20%` 前缀只达到约 `30%-35%`，而 `40%` 直接跃升到约 `90%`，这不是偶然波动，更像是训练分布和测试分布之间的错位。

原因主要来自当前训练配置：

- `online_train_prob = 0.6`
- `online_min_prefix_ratio = 0.4`
- `min_prefix_len = 64`

它们共同决定了：

- 不是每个训练样本都会被截断；
- 被截断的样本，其最短前缀通常也在 `40%` 左右；
- 因此模型几乎没有被要求在 `10%-20%` 这样的极短前缀上稳定工作。

换句话说，这个模型学到的是：

- “接触中段后尽快做出稳定判断”

而不是：

- “只接触一点点就立即判断”

因此，`40%` 前后的性能断崖，本质上是训练目标有意塑造出的结果，而不是单纯推理阶段的随机失误。

### 6.3 Gate 随进度变化的含义

`gate_score` 在 `10%-40%` 前缀时约为 `0.71-0.73`，到 `60%+` 降到约 `0.64-0.65`。

这说明：

- 早期触觉证据不足时，模型更愿意利用视觉信息；
- 当触觉证据逐渐充足后，模型会自然降低视觉依赖；
- 这与 OOD 下“触觉主导”的目标行为是一致的。

### 6.4 如果目标是更早预测，应该怎么改

如果未来应用场景更重视“一触即知”式的超快响应，而不是 `40%-60%` 时再做稳定判断，那么训练策略需要显式向超短前缀倾斜。

优先级最高的改法有两类：

1. 下调前缀训练下限

- 例如把 `online_min_prefix_ratio` 从 `0.4` 降到 `0.1` 或 `0.2`
- 让模型在训练中频繁见到 `10%-20%` 的短前缀

2. 改前缀采样分布

- 不是在 `[min_prefix, full_length]` 上近似均匀采样
- 而是改成偏向短前缀的分布，例如指数型、平方根型，或手工增加短前缀权重

如果这样做，通常会带来一个典型 trade-off：

- 超早期精度会上升；
- 但完整序列或中后期的最终精度可能略有下降

因此更合理的做法通常不是直接替换当前配置，而是额外训练一个：

- **early-response 强化版**

专门服务于“极早期预测”场景。

## 8. Multi-Seed 稳定性

为了确认 `fusion_gating_online_v2` 不是单个 seed 的偶然结果，进一步使用 `5` 个 seed 做了复现实验：

- seeds: `42, 123, 456, 789, 2024`
- 汇总文件：`visuotactile/outputs/fusion_gating_online_v2_multiseed/meta/multi_seed_summary_gating_online_v2.json`

### 8.1 Multi-Seed 汇总结果

| Split | Mass | Stiffness | Material | Avg Acc | Avg Gate |
| --- | ---: | ---: | ---: | ---: | ---: |
| `test` | `100.00 ± 0.00%` | `100.00 ± 0.00%` | `99.12 ± 1.76%` | `99.71 ± 0.59%` | `0.622 ± 0.046` |
| `ood_test` | `100.00 ± 0.00%` | `95.17 ± 7.61%` | `97.67 ± 3.43%` | `97.61 ± 3.68%` | `0.589 ± 0.072` |

### 8.2 结果解读

这组结果说明：

1. `fusion_gating_online_v2` 的优秀表现不是单个 seed 偶然撞出来的；
2. 分布内结果几乎满分，且方差很小；
3. OOD 结果虽然有一定波动，但整体仍然非常强；
4. gate 在多 seed 下仍保持中等水平，没有重新塌陷到 `0.9+` 的视觉常开状态。

从任务层面看：

- `mass` 基本已经稳定解决，`5` 个 seed 全是 `100%`；
- `material` 也非常稳；
- 真正的波动主要来自 `stiffness`，尤其是 OOD 下个别 seed 会拉高整体方差。

因此，当前这套配置的主要不确定性不在“模型会不会崩”，而在：

- OOD 下硬度任务会不会因为 seed 差异出现几个百分点到十个百分点的波动。

但即使考虑这种波动，整体平均 `97.61 ± 3.68%` 的 OOD 表现，仍然足以把它当作当前主力在线基线。

## 9. 综合判断

`fusion_gating_online_v2` 的优点非常明确：

1. 保住了离线 gating 模型在 OOD 上的强表现；
2. 通过前缀训练获得了不错的 anytime prediction 能力；
3. OOD 下模态使用模式健康，主要依赖触觉；
4. gate 没有饱和到无意义的极端，而是随前缀进度动态调整；
5. 多 seed 下仍保持很强的平均性能，说明复现性较好；
6. 训练收敛很快，工程上容易复现。

它的主要限制也同样明确：

1. 不是严格 streaming / causal model；
2. 每次到新前缀都要重新跑一遍模型；
3. 超早期 `10%-20%` 前缀性能仍然偏低；
4. 如果部署场景要求非常低延迟、非常高更新频率，那么重复整段前缀推理会带来额外计算成本。

## 10. 部署意义

如果部署目标是：

- 抓取进行到 `n%` 时进行一次预测；
- 再在更后面时刻重新判断一次；

那么这版已经是可以部署的。

推荐的使用方式是：

- 在 `40%` 左右给出第一次较可靠预测；
- 在 `60%` 左右进行确认；
- 如果需要极高置信度，则继续使用更长前缀。

按当前结果看：

- `40%` 已经可以达到约 `90.56%`
- `60%` 已经达到约 `99.72%`

因此它在工程上已经具备较强的在线决策价值。

---

## 11. 最终结论

`fusion_gating_online_v2` 是一个非常成功的“前缀在线版 gating 模型”。

它的本质不是“真正增量缓存式 streaming”，而是：

- **在给定当前触觉前缀的前提下，能够高质量地在线判断**

如果项目当前目标是：

- 在抓取过程中尽早做决策；
- 同时保持高 OOD 泛化能力；

那么这版可以视为当前最强、最实用的在线部署基线之一。
