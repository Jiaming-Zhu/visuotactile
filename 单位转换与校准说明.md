# 单位转换与校准说明

## 问题概述

在将 PyBullet IK 解写入真实机械臂时，涉及三层单位转换：
1. **PyBullet IK 输出**：弧度（rad）
2. **LeRobot 输入**：度数（°）或原始步进值
3. **舵机接收**：原始步进值（0-4096）

关键问题：**LeRobot 有两种模式（有校准/无校准），接受的输入格式不同！**

---

## 一、舵机的三种表示方式

### 1. 原始步进值（Raw Position）
- **范围**：0 到 4095（STS3215，12位分辨率）
- **含义**：舵机内部编码器的绝对位置
- **特点**：
  - 每个舵机的零位不同（可能是 2048、56734 等任意值）
  - 不同型号的分辨率不同
  - 没有统一的参考坐标系

```python
# 例如两个舵机的零位可能完全不同
motor1_zero = 2048  # 舵机1的零位
motor2_zero = 56734  # 舵机2的零位（绝对不同！）
```

### 2. 校准后的度数（Calibrated Degrees）
- **范围**：-180° 到 180°
- **含义**：以机械零位为中心的角度
- **特点**：
  - 所有舵机统一参考坐标系
  - 0° 表示机械零位（校准定义）
  - 可以处理旋转方向反转

```python
# 校准后，所有舵机的零位都是 0°
motor1_calibrated = 0.0  # 舵机1的零位
motor2_calibrated = 0.0  # 舵机2的零位（统一！）
```

### 3. PyBullet 的弧度（Radians）
- **范围**：-π 到 π（约 -3.14 到 3.14）
- **含义**：关节角度的标准数学表示
- **特点**：
  - 符合物理仿真习惯
  - 需要转换为度数才能用于 LeRobot

```python
# PyBullet IK 输出
ik_solution = [0.785, -1.571, 2.356]  # 弧度
# 对应的度数
degrees = [45.0, -90.0, 135.0]  # 度
```

---

## 二、LeRobot 的两种模式

### 模式 1：有校准模式（Calibrated Mode）

**使用场景**：通过 `robot.connect()` 连接，自动运行校准

**写入流程**：
```python
# 用户代码
motors.write("Goal_Position", [45.0, -90.0, 135.0])  # 输入：度数

# LeRobot 内部（feetech.py line 1181-1182）
if data_name in CALIBRATION_REQUIRED and self.calibration is not None:
    values = self.revert_calibration(values, motor_names)
    # 自动转换：度数 → 原始步进值

# 发送到舵机
# 舵机接收：原始步进值（如 [2500, 1596, 3500]）
```

**转换公式**（`revert_calibration` 的逻辑）：
```python
# 1. 度数 → 分辨率范围
values[i] = values[i] / 180.0 * (resolution // 2)
# 例：45° → 1024（对于 resolution=4096）

# 2. 减去零位偏移
values[i] -= homing_offset
# 例：1024 - (-1476) = 2500

# 3. 应用旋转方向
if drive_mode:
    values[i] *= -1
```

### 模式 2：无校准模式（Uncalibrated Mode）

**使用场景**：直接创建 `MotorsBus`，不运行校准

**写入流程**：
```python
# 用户代码
motors.write("Goal_Position", [2500, 1596, 3500])  # 输入：原始步进值

# LeRobot 内部（feetech.py line 1181-1182）
if data_name in CALIBRATION_REQUIRED and self.calibration is not None:
    # self.calibration is None，跳过转换！

# 发送到舵机
# 舵机接收：原始步进值（如 [2500, 1596, 3500]）
```

**无转换**：直接使用原始步进值

### 当前实现的转换链（✅ 正确）


## 四、为什么需要校准？

### 问题 1：零位不统一
```python
# 不同舵机装配后的零位位置不同
motor1_raw_zero = 2048   # 舵机1的物理零位
motor2_raw_zero = 56734  # 舵机2的物理零位（差异巨大！）

# 校准后统一为 0°
motor1_calibrated_zero = 0.0
motor2_calibrated_zero = 0.0
```

### 问题 2：旋转方向不一致
```python
# 有些舵机装配方向相反
motor1_direction = 1   # 正向
motor2_direction = -1  # 反向（装反了）

# 校准后可以纠正方向
calibration["drive_mode"] = [0, 1]  # 0=正向，1=反向
```

### 问题 3：分辨率不同
```python
# 不同型号的分辨率不同
sts3215_resolution = 4096   # FeeeTech STS3215
xl330_resolution = 4096     # Dynamixel XL330
mx64_resolution = 4096      # Dynamixel MX-64

# 校准统一使用度数，自动处理分辨率差异
```

### ✅ 当前实现已正确处理

- [x] PyBullet IK 输出：弧度
- [x] 转换为度数：`np.rad2deg()`

- [x] 发送到舵机：步进值

### ⚠️ 需要注意的事项

1. **必须运行校准**：使用 `robot.connect()` 时会自动运行校准
2. **校准文件必须存在**：否则会进入交互式校准流程
3. **只控制前 6 个关节**：`joint_angles_rad[:6]`，不包括夹爪
4. **单位必须是度数**：`write("Goal_Position", degrees)`

## 六、常见错误与排查

### 错误 1：写入弧度而非度数

```python
# ❌ 错误：直接写入弧度
ik_solution = [0.785, -1.571, 2.356]  # 弧度
motors.write("Goal_Position", ik_solution)
# 舵机会移动到 0.785°（非常小的角度！）

# ✅ 正确：转换为度数
ik_solution_deg = np.rad2deg(ik_solution)  # [45.0, -90.0, 135.0]
motors.write("Goal_Position", ik_solution_deg)
```

### 错误 2：在无校准模式下使用度数

```python
# ❌ 错误：无校准模式下写入度数
motors = FeetechMotorsBus(config)
motors.connect()
# 假设没有校准文件，self.calibration = None
motors.write("Goal_Position", [45.0, -90.0])
# 舵机会移动到步进值 45 和 -90（错误！）

# ✅ 正确：使用原始步进值
motors.write("Goal_Position", [2500, 1596])
```

### 错误 3：混淆读取和写入的单位

```python
# 有校准模式下
pos_calibrated = motors.read("Present_Position")
# 返回：度数（如 [45.0, -90.0]）

motors.write("Goal_Position", pos_calibrated)
# 输入：度数（如 [45.0, -90.0]）
# LeRobot 自动转换为步进值后发送

# ✅ 单位一致，正确！
```

## 八、总结

### 核心要点

1. **PyBullet IK 输出**：弧度（rad）
2. **LeRobot 有校准模式输入**：度数（°）
3. **LeRobot 自动转换**：度数 → 步进值
4. **舵机接收**：步进值

## 九、参考代码

### 完整的写入函数（带注释）

```python
def write_to_real_robot(joint_angles_rad):
    """
    将 PyBullet IK 解（弧度）写入真实机械臂
    
    单位转换流程：
    1. PyBullet IK 输出：弧度（rad）
    2. 转换为度数：np.rad2deg()
    3. 输入到 LeRobot：度数（float32）
    4. LeRobot 自动转换：度数 → 步进值（revert_calibration）
    5. 发送到舵机：步进值
    
    Args:
        joint_angles_rad: PyBullet IK 解（弧度），shape=(n_joints,)
    """
    global real_robot, use_real_robot
    
    if not use_real_robot or real_robot is None:
        print("⚠️  真实机械臂未连接，跳过写入")
        return False
    
    try:
        # ===== 步骤 1：弧度 → 度数 =====
        # PyBullet 输出弧度，LeRobot 有校准模式接受度数
        joint_angles_deg = np.rad2deg(joint_angles_rad[:6])
        print(f"IK解（弧度）：{joint_angles_rad[:6]}")
        print(f"IK解（度数）：{joint_angles_deg}")
        
        # ===== 步骤 2：转为 PyTorch tensor =====
        # 与 manipulator.py 保持一致的数据格式
        goal_pos = torch.from_numpy(joint_angles_deg).float()
        
        # ===== 步骤 3：写入舵机 =====
        for name in real_robot.follower_arms:
            # （可选）安全检查
            if real_robot.config.max_relative_target is not None:
                present_pos = real_robot.follower_arms[name].read("Present_Position")
                present_pos = torch.from_numpy(present_pos)
                goal_pos = ensure_safe_goal_position(
                    goal_pos, present_pos, real_robot.config.max_relative_target
                )
            
            # 转为 numpy float32
            goal_pos_np = goal_pos.numpy().astype(np.float32)
            
            # 写入舵机（LeRobot 自动转换为步进值）
            real_robot.follower_arms[name].write("Goal_Position", goal_pos_np)
            print(f"✓ 已写入 {name} follower 臂（度数 → 自动转换为步进值）")
        
        return True
        
    except Exception as e:
        print(f"❌ 写入失败: {e}")
        return False
```
