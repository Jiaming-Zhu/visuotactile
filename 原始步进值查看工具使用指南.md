# 原始步进值查看工具使用指南

## 🎯 功能简介

新增 `raw` / `steps` 命令，可以**实时显示真实机械臂的原始步进值**（未经校准的舵机读数），并支持按 Enter 键保存当前位置。

这个功能主要用于：
- 🔍 **调试校准问题**：查看舵机的真实读数
- 📊 **记录零点位置**：手动摆放姿势后保存原始步进值
- 🧪 **验证舵机通信**：确认舵机是否正常工作
- 🛠️ **自定义校准**：收集数据用于手动计算校准参数

## 🚀 使用方法

### 基本用法

```bash
# 1. 启动程序
python interactive_control.py

# 2. 连接机械臂
> connect

# 3. 显示原始步进值
> raw
# 或
> steps
```

### 界面说明

进入实时显示模式后，你会看到：

```

================================================================================

按 Enter 保存 | 输入 'q' + Enter 退出

--------------------------------------------------------------------------------

    1 | shoulder_lift        |          993 |     -45.12°
    2 | elbow_flex           |         3061 |      78.23°

--------------------------------------------------------------------------------

已保存 0 个位置

💡 提示：按 Enter 保存当前位置，输入 'q' + Enter 退出
```

### 保存位置

当你想保存当前机械臂的姿势时：

1. **手动调整机械臂**到目标姿势（进入 raw 模式后力矩自动禁用）
2. **按保存键**：
   - **Linux/Mac**：按 Enter 键
   - **Windows**：按 's' 键或 Enter 键
3. 自动保存到 JSON 文件（立即保存，不会丢失）

保存的文件格式：
```json
[
  {
    "timestamp": "2025-01-30 15:23:45",
    "raw_steps": [2049, 993, 3061, 1978, 2133, 2046],
    "calibrated_angles": [0.00, -45.12, 78.23, -12.05, 5.02, 50.23],
    "motor_names": ["shoulder_pan", "shoulder_lift", "elbow_flex", "wrist_flex", "wrist_roll", "gripper"]
  },
  {
    "timestamp": "2025-01-30 15:24:10",
    ...
  }
]
```

### 退出

退出方式：
- **Linux/Mac**：输入 `q` + Enter 或按 Ctrl+C
- **Windows**：按 `q` 键（无需 Enter）或 Ctrl+C

退出时会显示保存摘要。

## 📊 数据说明

### 原始步进值

- **范围**：通常是 0 ~ 4095（Feetech 舵机）
- **含义**：舵机的实际物理位置（未经校准）
- **特点**：
  - 不同舵机的"零点"可能完全不同
  - 例如：舵机A的零点可能在 2048，舵机B的零点可能在 993

### 校准后角度

- **范围**：-180° ~ +180°（旋转关节）或 0% ~ 100%（夹爪）
- **含义**：经过校准转换后的统一角度
- **特点**：
  - 所有舵机的"0°"对应同一个物理姿势（Zero Position）
  - 便于跨机械臂使用

## 🎓 应用场景

### 场景1：记录 Zero Position（零位）

**目的**：找到校准文件中 `homing_offset` 的正确值

**步骤**：
```bash
1. 启动程序并连接机械臂
   > python interactive_control.py
   > connect

2. 手动将机械臂摆成标准零位姿势：
   - 水平伸直
   - 夹爪朝上
   - 夹爪闭合

3. 显示原始步进值
   > raw

4. 按 Enter 保存

5. 查看保存的 JSON 文件中的 raw_steps
   这就是你的零位对应的步进值！
```

**结果应用**：
```json
{
  "raw_steps": [2049, 993, 3061, 1978, 2133, 2046]
}
```

计算 `homing_offset`：
```python
homing_offset = 0 - raw_steps
             = [0, 0, 0, 0, 0, 0] - [2049, 993, 3061, 1978, 2133, 2046]
             = [-2049, -993, -3061, -1978, -2133, -2046]
```

### 场景2：对比多个姿势

**目的**：记录不同姿势的步进值，用于分析关节运动范围

**步骤**：
```bash
1. > raw

2. 手动摆放第一个姿势（如零位）
   按 Enter 保存

3. 手动摆放第二个姿势（如旋转位）
   按 Enter 保存

4. 手动摆放第三个姿势（如休息位）
   按 Enter 保存

5. 输入 'q' 退出
```

**结果分析**：
```json
[
  {
    "timestamp": "2025-01-30 15:23:45",
    "raw_steps": [2049, 993, 3061, 1978, 2133, 2046],
    "motor_names": [...]
  },
  {
    "timestamp": "2025-01-30 15:24:10",
    "raw_steps": [3073, 993, 2051, 2980, 1071, 2046],
    "motor_names": [...]
  }
]
```

分析关节 0 的运动：
```
姿势1: 2049
姿势2: 3073
差值: 1024 步 ≈ 90° (1024 / 2048 × 180°)
```

### 场景3：验证校准效果

**目的**：对比原始步进值和校准后角度，验证校准是否正确

**步骤**：
```bash
1. > raw

2. 观察两列数据：
   - 原始步进值：舵机的真实读数
   - 校准后角度：经过 homing_offset 和 drive_mode 转换后的值

3. 验证零位：
   当机械臂在零位姿势时，校准后角度应该接近 0°
```

**示例分析**：
```
索引 | 舵机名称         | 原始步进值   | 校准后角度  
-----|------------------|-------------|------------
  0  | shoulder_pan    |       2049  |      0.00°  ✓ 正确
  1  | shoulder_lift   |        993  |    -45.12°  ⚠️ 不是0°，说明不在零位
```

### 场景4：故障诊断

**目的**：检查舵机是否正常工作

**步骤**：
```bash
1. > raw

2. 手动移动机械臂

3. 观察原始步进值是否实时变化：
   - ✓ 变化正常：舵机通信正常
   - ✗ 不变化：可能是舵机故障或连接问题
   - ✗ 跳变异常：可能是电缆接触不良
```

## 🔧 技术细节

### 读取原始步进值的方法

代码实现：
```python
# 直接读取原始值（绕过校准）
for motor_name in motor_bus.motor_names:
    motor_idx, motor_model = motor_bus.motors[motor_name]
    raw_val = motor_bus.read_with_motor_ids(
        [motor_model], 
        [motor_idx], 
        "Present_Position"
    )
```

与校准读取的区别：
```python
# 校准读取（自动转换为角度）
calibrated_angles = motor_bus.read("Present_Position")
# ↓ 内部调用 apply_calibration()
# 步进值 → 角度

# 原始读取（不经过校准）
raw_steps = motor_bus.read_with_motor_ids(...)
# 直接返回步进值
```

### 非阻塞输入处理

**Linux/Mac** - 使用 `select` 模块：
```python
import select
import sys

# 检查是否有输入（等待 0.3 秒）
if select.select([sys.stdin], [], [], 0.3)[0]:
    user_input = sys.stdin.readline().strip()
```

**Windows** - 使用 `msvcrt` 模块：
```python
import msvcrt

# 检查是否有按键
if msvcrt.kbhit():
    key = msvcrt.getch()
    if key == b's':  # 保存
        save_position()
```

## 💡 使用技巧

### 技巧1：快速记录零点

```bash
1. 连接机械臂，禁用力矩（让机械臂可以手动移动）
   > connect
   
2. 手动摆成零位姿势
   - 水平伸直
   - 夹爪朝上
   
3. 记录步进值
   > raw
   > [按 Enter]
   > q
   
4. 查看保存的 JSON 文件
```

### 技巧2：对比仿真和真实

```bash
1. 在仿真中回零
   > joint 0 0 0 0 0
   
2. 查看真实机械臂的步进值
   > raw
   
3. 对比：
   - 如果校准正确，真实机械臂应该也在零位
   - 原始步进值应该接近 homing_offset
```

### 技巧3：验证单个关节

```bash
1. 进入原始步进值显示
   > raw
   
2. 只移动某一个关节（如关节2）
   
3. 观察只有关节2的步进值变化
   - 如果其他关节也变化：可能有机械耦合
```

## ⚠️ 注意事项

1. **必须先连接机械臂**
   ```bash
   > connect    # 必须先执行
   > raw        # 才能使用
   ```

2. **自动禁用力矩** ⭐ 重要
   - 进入 raw 模式时，自动禁用舵机力矩
   - 这样可以**手动移动机械臂**记录不同姿势
   - 退出后保持禁用状态（安全考虑）

3. **刷新频率**
   - 当前设置为每 0.3 秒刷新一次
   - 如果需要更快刷新，可以修改代码中的 `time.sleep(0.3)`

4. **跨平台支持** ✅
   - **Linux/Mac**：按 Enter 保存，输入 'q' + Enter 退出
   - **Windows**：按 's' 或 Enter 保存，按 'q' 退出
   - 两个平台都完全支持保存功能！

5. **立即保存**
   - 每次按保存键都会立即写入文件
   - 不用担心数据丢失

6. **文件命名**
   - 每次保存都会更新同一个文件
   - 退出时生成最终文件
   - 不会覆盖之前的记录（时间戳不同）

7. **步进值范围**
   - Feetech 舵机：0 ~ 4095
   - 如果看到负数或超过 4095：可能有溢出或连续旋转
